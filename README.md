# 2. システム概要

## 2.1 システム目的

本システムは、学校内における生徒の名札着用確認を自動化するためのシステムである。
Raspberry Pi・複数カメラ・AI画像判定を利用し、生徒の名札着用有無をリアルタイムで判定する。

本システムにより以下を実現する。

* 風紀委員の負担軽減
* 名札確認漏れの防止
* 判定ログの自動保存
* 複数カメラによる判定精度向上

# 7. 処理フロー設計

## 7.1 システム処理フロー

```text
① 人感センサー検知
        ↓
② 複数カメラ同時撮影
        ↓
③ 各画像をAI判定
        ↓
④ 判定結果統合
        ↓
⑤ 結果表示
        ↓
⑥ ログ保存
        ↓
⑦ 必要に応じLINE通知
```

## 7.2 判定ロジック

| 条件          | 最終判定  |
| ----------- | ----- |
| 1台以上が「名札あり」 | 名札あり  |
| 全台「名札なし」    | 名札なし  |
| 撮影失敗あり      | 再撮影実施 |

### 判定アルゴリズム

```python
if any(camera_result == "あり"):
    final_result = "名札あり"
else:
    final_result = "名札なし"
```

# 11. エラー処理設計

## 11.1 エラー一覧

| エラー     | 原因       | 対応       |
| ------- | -------- | -------- |
| カメラ接続失敗 | USB切断    | 再接続・通知   |
| AI判定失敗  | API通信エラー | 再送信      |
| 撮影失敗    | 照明不足     | 再撮影      |
| SD容量不足  | ログ肥大化    | 古いログ削除   |
| ネットワーク断 | Wi-Fi障害  | ローカル待機処理 |

## 11.2 障害時動作

* カメラ1台故障時でも残りカメラで継続
* API失敗時は最大3回リトライ
* ログ保存失敗時はメモリ一時保存

# 15. 開発体制

## 15.1 チーム構成

本開発では、学生メンバーによる開発を前提とし、知識共有・レビュー強化・作業負荷分散を目的として「2人1組のペア開発方式」を採用する。

PMを除く6名を3チームに分け、各チームが1つの機能群を担当する。

| 役割           | 人数 | 担当領域              |
| ------------ | -- | ----------------- |
| PM           | 1  | 進捗管理・レビュー・学校調整    |
| ハードウェアチーム    | 2  | カメラ・センサー・GPIO制御   |
| AI/バックエンドチーム | 2  | AI判定・統合ロジック・API連携 |
| UI/テストチーム    | 2  | 画面表示・ログ・テスト・通知機能  |

合計：7名

### ペア開発方式

| チーム          | 開発スタイル                     |
| ------------ | -------------------------- |
| ハードウェアチーム    | 1人が実装、1人が配線・動作確認を担当        |
| AI/バックエンドチーム | 1人がAPI実装、1人が判定ロジック・レビューを担当 |
| UI/テストチーム    | 1人がUI作成、1人がテスト・ログ確認を担当     |

### ペア開発の目的

* 学習しながら開発を進められる
* コードレビューをリアルタイムで実施できる
* 属人化を防止できる
* 問題発生時に相談しやすい
* 学生同士で知識共有ができる

## 15.2 担当詳細

| 担当          | 人数 | 作業内容                        |
| ----------- | -- | --------------------------- |
| PM          | 1  | WBS管理、レビュー、進捗確認、学校との調整      |
| ハードウェアペア    | 2  | カメラ接続、GPIO制御、PIRセンサー、ラズパイ設定 |
| AI/バックエンドペア | 2  | Claude API連携、画像判定、統合ロジック実装  |
| UI/テストペア    | 2  | 画面作成、ログ機能、LINE通知、テスト実施      |

### チーム間連携ルール

* GitHubを用いてソースコードを共有する
* Pull Requestベースでレビューを実施する
* 毎週1回、PM主導で進捗確認会を行う
* 各ペアは週次で作業内容を記録する
* 不明点はチーム内で確認後、PMへ相談する

---|---|
|PM|WBS管理、レビュー、進捗確認|
|メンバーA|カメラ制御実装|
|メンバーB|PIRセンサー制御|
|メンバーC|Claude API連携|
|メンバーD|AI判定ロジック|
|メンバーE|UI画面作成|
|メンバーF|テスト・ログ・運用設計|
